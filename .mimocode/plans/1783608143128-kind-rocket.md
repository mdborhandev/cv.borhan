# Fix: Broken Layout & Responsive Design

## Problem
The previous merge copied `tw-` prefixed Tailwind classes from the jrdev template into `index.html`. The jrdev template uses a **custom Tailwind build** with a `tw-` prefix, but this project uses `cdn.tailwindcss.com` which generates **standard Tailwind classes** (no prefix). Result: ~90% of layout/utility classes are invisible to the CDN — the layout is completely broken.

## Root Cause
Every `tw-` prefixed class in the HTML is dead. Examples:
- `tw-flex` → should be `flex`
- `tw-h-full` → should be `h-full`
- `tw-w-[400px]` → should be `w-[400px]`
- `tw-overflow-hidden` → should be `overflow-hidden`
- `max-lg:tw-flex-col` → should be `max-lg:flex-col`

## Files to Modify

### 1. `index.html` — Replace all `tw-` prefixed classes with standard Tailwind

Strip the `tw-` prefix from every utility class. Also fix structural issues:
- `<main>`: `tw-flex tw-h-full tw-w-full tw-overflow-hidden max-lg:tw-flex-col tw-h-screen` → `flex h-screen w-full overflow-hidden max-lg:flex-col`
- `<aside>`: `tw-flex tw-h-full tw-w-[400px] tw-flex-col tw-place-items-center tw-gap-4 tw-p-6 max-lg:tw-w-full` → `flex h-full w-[400px] flex-col items-center gap-4 p-6 max-lg:w-full`
- Right panel div: `tw-h-full tw-w-full tw-overflow-hidden tw-flex tw-flex-col` → `h-full w-full overflow-hidden flex flex-col`
- Header: `tw-relative tw-flex tw-w-full tw-overflow-hidden tw-p-2 max-lg:tw-h-[150px] max-lg:tw-flex-col lg:tw-h-[80px]` → `relative flex w-full overflow-hidden p-2 max-lg:h-[150px] max-lg:flex-col lg:h-[80px]`
- Tab bar: similar prefix removal
- All tab content sections: similar prefix removal
- Tab content wrapper: `tw-overflow-y-auto tw-flex-1` → `overflow-y-auto flex-1`
- Footer: similar prefix removal

### 2. `style.css` — Minimal cleanup

- Ensure `.sidebar-panel` background is correct
- Add responsive styles for sidebar collapse on mobile

### 3. `script.js` — No changes needed
